    How the scoring works

    Each use case is scored on 7 criteria (1–5). Criteria are grouped into two axes:

    • Value = weighted average of Strategic alignment (×1.2), Financial impact (×1.2), Customer impact (×1.0).
    • Feasibility = weighted average of Data readiness (×1.2), Technical complexity (×1.0, inverted), Risk / governance burden (×1.0, inverted), Time to value (×0.8, inverted).

    Composite score = (Value × 0.6 + Feasibility × 0.4) × 20, mapped 0–100. Quadrant assignment uses a cut-off at 3.0 on each axis (the midpoint of 1–5). Tier thresholds: 0–20 Kill, 20–40 Defer, 40–60 Pilot, 60–80 Scale, 80+ Flagship. The recommendation engine maps each quadrant and the lowest-scoring criteria to pre-authored guidance — there is no LLM at runtime.
